Understanding LLMs: From Pretraining to Post‚ÄëTraining#

At the pretraining stage, an LLM (large language model) acquires knowledge from the internet and stores it in the parameters of a neural network. This produces what we call a base model‚Äîa token autocompleter. Then we apply supervised fine‚Äëtuning (SFT) on curated conversation data. Pretraining#

All pages on the internet are filtered and combined into one massive text corpus.
This text is then tokenized into a sequence of finite symbols (tokens).

Neural Network Training#

The model learns the statistical relationships between tokens‚Äîthat is, how likely each token is to follow the ones before it in a sequence. During training, we fix a context window size (also called the context length). Within this window, the model looks at a sequence of tokens and predicts what comes next.

  • Input: a variable‚Äëlength sequence of tokens (up to the context window size)
  • Output: a probability distribution over the next possible token

Inference#

When you talk to ChatGPT, the model is not learning anymore‚Äîit‚Äôs inferring:

  1. Take your text as input.
  2. Predict the next token.
  3. Append it to the context.
  4. Repeat step by step until a full response is generated.

Post‚ÄëTraining on Conversations (Mimicking an Expert Solution with SFT)#

After pretraining, the base model is further refined through Supervised Fine‚ÄëTuning (SFT). In this phase, human labelers create high‚Äëquality example conversations.
These conversations are fed back into the base model to teach it how to respond in a more helpful, structured, and human‚Äëlike way. What you receive from OpenAI is essentially a statistical simulation of those human labelers:

  • If your query is similar to examples in the training data, the response will closely match those patterns.
  • If your query is unlike anything in the training data, the model relies on its broader, pretrained knowledge from the internet and prior conversations to generate a best‚Äëeffort answer.

Hallucinations#

When the model lacks relevant knowledge, it may still respond with confident but incorrect information‚Äîthis is known as a hallucination.

How can we reduce this?

Leverage working memory (context).

This image explains the analogy ‚Äî knowledge in parameters is like vague recollection, while knowledge in the current context window is like working memory. The knowledge stored in the model‚Äôs parameters is like a vague recollection from pretraining. In contrast, the information you include in the current context window‚Äîyour prompt and any supplied data‚Äîacts as the model‚Äôs working memory. This context is directly accessible and far more reliable for generating accurate responses.

Models need tokens to think#

Each token prediction in an LLM uses only a limited amount of computation. If you try to force the model to perform all reasoning in a single token, it may fail or produce errors. That‚Äôs why ChatGPT often works step by step‚Äîit spreads computation across multiple tokens and multiple passes through the network.

Be mindful when asking ChatGPT to do arithmetic or counting.

Post‚ÄëTraining with Reinforcement Learning (Verifiable Domains)#

The image below illustrates an analogy between how ChatGPT learns and how a child learns. Reinforcement learning (RL) is used to further refine models. For example, AlphaGo used RL to discover new ways of playing Go. Could RL in LLMs create new ways of thinking? It‚Äôs an exciting possibility. In reinforcement learning (RL), we provide only the problem statement (the environment and rules) and the final outcomes (rewards or answers), but not the full step‚Äëby‚Äëstep solution.
The model or agent learns by exploring different actions, observing the results, and gradually improving its strategy based on feedback, rather than being explicitly shown how to solve the problem. RLHF (Non‚ÄëVerifiable Domains)#

In some tasks, it‚Äôs easy to verify the answer‚Äîlike checking that 1 + 1 = 2.
But what if the task is subjective, such as writing a good joke? There‚Äôs no single ‚Äúcorrect‚Äù answer. Scoring jokes directly with humans is very resource‚Äëintensive.
To make this feasible, researchers create a reward model that tries to mimic human preferences:

  1. Humans are asked to rank or order multiple outputs (for example, several jokes).
  2. These rankings train a reward model that estimates what a ‚Äúgood‚Äù output looks like.
  3. The LLM then uses this reward model to guide its responses.

Unlike standard reinforcement learning in verifiable domains, RLHF cannot run indefinitely because the model may eventually game the reward function, generating edge‚Äëcase outputs that score well but make little sense to humans.
